Czym jest dystrybucja Linuksa?

Oprogramowanie
  • kernel
  • biblioteki
  • aplikacje
  • instalator
  • narzędzia konfiguracyjne
  • poprawki/łaty
Nie tylko kod
  • fani, użytkownicy
  • gadżety
  • ikony, tapety i inne wodotryski
  • wsparcie techniczne
  • listy dyskusyjne
  • systemy śledzenia błędów
  • marketing
  • dokumentacja
  • testowanie

Różnice pomiędzy dystrybucjami

  • sposób instalacji
  • sposób uruchamiania: z dysku po instalacji, liveCD/DVD, USB, flash
  • ilość pakietów: Debian Mint Romeo (41992), Ubuntu 12.4 (38669), Red Hat 5.2 (300 w 1998 roku)
  • kategorie oprogramowania
  • zarządzanie pakietami: deb (apt-get), rpm (yum, zypper), pacman, tgz
  • aktualizacja systemu
  • filozofia/religia?

Krótki przegląd dystrybucji

  • SLS (1992) - jedna z pierwszych dystrybucji
    • Slackware (1993) - przejście z a.out na ELF - około 3000 pakietów
    • Debian (1993) - rozwijany przez ochotników z całego świata
      • Ubuntu (2004) - rozwijana przez Canonical
        • Mint (2006) - zawiera pakiety których nie ma w Ubuntu ani Debianie
      • Knoppix (2000) - LiveCD
  • Tiny Core Linux (2009) - mini dystrybucja (10 MB), urządzenia loop
  • Arch (2002) - bazuje na najnowszych wersjach oprogramowania, bez dodatkowych patchy, bardzo proste skrypty do kompilacji pakietów
  • Gentoo (2002) - kompilacja prawie całego systemu z kodu źródłowego
  • Linux from scratch (1999) - dokumentacja, jak zbudować samemu własny system
    • Beyond Linux From Scratch - dodatkowe pakiety
    • Automated Linux From Scratch
    • Cross Linux From Scratch
    • Hardened Linux From Scratch - security
    • distrowatch.com - baza informacji o dystrybucjach Linuksa i BSD

Dystrybucje korporacyjne

  • Red Hat (1994) / Fedora (2003)
  • SuSe/SLES (1994) / OpenSuse (2006)
