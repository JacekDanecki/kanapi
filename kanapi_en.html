About KaNaPi

KaNaPi is:
  • a Linux distribution
  • free and open sourced
  • educational operating system
  • based on Linux kernel and free/open source software
  • created from scratch
  • distributed as build system in source files
  • distributed as disk images
  • distributed as docker images
  • compiled for 3 CPU architectures: x86_64, x86, arm
  • simple in use
  • built using only few commands

KaNaPi and other Linux distributions

  • KaNaPi contains only selected packages mainly for use at home.
  • Each package is installed in separate directory (similar to GoboLinux, but in KaNaPi each version of specific package is located in separate directory in /kanapi_0.9/x86_64/packages. For example:
    • /kanapi_0.9/x86_64/package/gcc-4.9.4
    • /kanapi_0.9/x86_64/package/gcc-5.5.0
    • /kanapi_0.9/x86_64/package/gcc-6.4.0
    • /kanapi_0.9/x86_64/package/gcc-7.2.0
  • /etc directory contains only few files. Configuration for specific package is available in package directory or in XDG directories. 
    # ls  /etc  
group  modprobe.conf  netconfig      OpenCL  resolv.conf
hosts  mtab           nsswitch.conf  passwd  services
  • To start system kernel executes simple /init script written in bash. There is not systemd, nor any other service manager.
  • There is only one user: kanapi with admin privileges, passwords are not needed, so you don't have to login
  • There are additional text consoles executed by openvt. If you need more, screen is provided
  • There is not GDM, lightgdm, nor any other display manager to login to GUI
  • i3 is provided as main window manager. Scripts for vtvm or ratpoison are also provided. Support for XFCE was dropped.
  • To run applications compiled outside KaNaPi linker in /lib64/ld-linux-x86-64.so.2 points at /kanapi_0.9/x86_64/packages/glibc-2.23/lib64/ld-linux-x86-64.so.2
  • To simplify build process:
    • /usr/local/include directory contains links to all headers from packages installed in the system
    • /kanapi_0.9/x86_64/packages/libs directory contains links to all libraries from packages installed in the system
  • pkg-config is main source of information about installed packages, and parameters needed for compilation
  • pkg script helps to synchronize binary packages from SourceForge
  • pkg_kanapi is low level script to work with packages sources, configuration, installation and verification
  • Build system is based on Makefile files and bash scripts
  • KaNaPi provides scripts to run another Linux distributions or only one application inside chroot or qemu
